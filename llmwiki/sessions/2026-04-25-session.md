## Major Decisions Made

### Built-In Materials

Concrete and steel presets were added/expanded for Turkish, Eurocode, and ACI-style ready classes.
The steel preset and concrete preset were separated so changing one does not unnecessarily reset the other.

Important behavior:
- Concrete preset selection can set default steel preset by code family.
- Manual strength edits should only mark the relevant preset as custom.
- Code-mode filtering was added for preset visibility.

### M-Phi Model Separation

For nonlinear M-phi, TS500/TBDY code modes now use a continuous material model path rather than the
TS500 equivalent rectangular block. The code-check/PMM path remains code-oriented.

Rationale:
- Equivalent rectangular stress block is appropriate for nominal/code capacity checks.
- Moment-curvature response should use continuous stress-strain behavior.
- TS500/TBDY M-phi uses Mander core+cover and unreduced section strengths for response analysis.

### M-Phi Diagnostics And UX

M-phi was expanded into a more explicit nonlinear section analysis view.

Added or changed:
- requested/solved/displayed point counts
- root-not-found and near-miss diagnostics
- phi gap reporting
- event markers such as cracking, steel yield, peak, cover limit, and final point
- table event labels
- monitoring points for selected M-phi step
- table hover/click sync with yellow markers on both M-phi and strain diagrams
- toast warnings for M-phi errors
- PMM envelope guard for unrealistic axial loads, relaxed up to roughly 4x envelope before attempting solve

Known behavior:
- If an axial load is beyond the physical solvability of the section, the UI may allow the attempt but show that no valid M-phi points were produced.
- Very high compression loads can still expose numerical branch-selection issues in the solver. The next serious numerical improvement should be in `assembly/index.ts`, around neutral-axis root branch selection.

### Plotly Lifecycle

Several bugs came from clearing a Plotly host and then trying to reuse it.

Current pattern:
- Use `renderPlotlyFigure(plotly, host, data, layout, config)`.
- It chooses `Plotly.react` only when a Plotly figure exists, otherwise `Plotly.newPlot`.
- Use `purgePlotlyFigure(host)` when clearing outputs.
- Do not write persistent loading text into plot hosts; use `aria-busy` and `aria-label`.
- `purgePlotlyFigure` uses `(globalThis as any).Plotly` to avoid TypeScript global-name build errors.

The Railway error fixed in `32a42e2` was:

```text
src/main.ts(8118,27): error TS2304: Cannot find name 'Plotly'.
```

### 3D PMM Panel

The right side of the 3D PMM panel could remain blank because Plotly kept a stale narrow width.

Current fix:
- `renderPlot3d()` reads the host width/height and passes it to Plotly layout.
- After rendering, it calls `Plots.resize(host)`.
- CSS sets `.viz3d-split .plot3d` to `min-width: 0` and `justify-self: stretch`.
